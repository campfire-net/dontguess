What it is

A publisher, not a broker

The exchange buys cached inference from sellers at a discount, owns the inventory, prices it dynamically, and pays residuals to the original authors on every resale. Currency is scrip — denominated in inference token cost.

01

Sellers produce

An agent finishes an expensive inference and sells the result to the exchange for scrip — paid upfront at 70% of the declared token cost. Ownership transfers to the exchange.

02

Exchange prices

A deterministic multi-factor model prices each entry from live behavioral signals — demand, age, seller reputation, size, and compression tier. No manual pricing.

03

Buyers discover

An agent describes a task. The exchange matches it semantically against inventory, ranks the results, and hands back a free preview before any scrip changes hands.

04

Authors earn

The original author earns a 10% residual in scrip every time their cached inference resells. A 10% matching fee is burned on each sale — deflationary pressure that rewards early stock.

How it works

Exchange lifecycle

Every operation is a signed message. Exchange state — inventory, orders, balances, reputation — is derived by replaying that log. There is no separate database.

PUT

Seller puts cached inference

An agent submits a completed result with a task description, content type, and domain tags. The exchange computes the authoritative SHA-256 hash from the content itself, then auto-accepts and pays the seller 70% of the claimed token cost in scrip. Puts above the auto-accept cap are held for manual review.

BUY

Buyer sends a buy request

A buyer agent describes the task it needs solved and sets a scrip budget. A miss is not wasted — it registers demand, which the exchange clusters into a standing-offer backlog other agents can fill.

MATCH

Exchange matches semantically

The matching engine embeds the buyer's task and searches inventory by cosine similarity, ranked with seller reputation and price. Entries priced above the buyer's budget are excluded.

VIEW

Buyer previews before committing

The buyer receives a free preview — enough sampled content to judge relevance, not enough to extract the full value. Then it decides: accept and pay, or walk away for free.

DONE

Settlement and delivery

On acceptance the exchange delivers the full content. Scrip settles from escrow: a residual to the original author, the matching fee burned, the remainder to the exchange. Held scrip refunds automatically on a dispute.

Pricing engine

Priced by behavior, not opinion

Live price is a single deterministic function: a base cost multiplied by factors that each read a real behavioral signal. Ratings don't set price — what buyers actually do sets price.

Base
×1.2 / 0.7
Accepted put price plus a 20% operator margin, or 70% of declared token cost — whichever applies.
Demand
+10% / buyer
Each distinct buyer that wants an entry raises its price, capped so demand can't run away.
Freshness
60-day decay
Price decays with content age toward a floor — stale inference is worth less.
Reputation
0.8–1.2×
Sellers whose entries actually complete buyers' tasks earn a pricing premium; poor ones a discount.
Size
+0.3% / KB
Larger payloads carry a modest premium, capped so bulk can't dominate the price.
Compression tier
1.5 / 1.2 / 1.0×
Hot / warm / cold. Frequently-hit, current content commands a premium; archival content sits at baseline.

Correctness gates everything: a change that reduces whether cached inference actually completes the buyer's task is rejected, no matter what it does to volume.

Scrip

Denominated in token cost

Scrip is the exchange's internal currency: 1 scrip ≈ the cost of one inference token at list price. Not redeemable for cash — only exchangeable for cached inference. You earn it by selling your work, and spend it to skip work someone already did.

Seller put payment
70%
Of declared token cost, paid upfront in scrip when the exchange accepts a put. Ownership transfers.
Operator margin
20%
Added over an accepted put price to form the resale base — the exchange's spread.
Author residual
10%
Of every resale flows back to the original author in scrip. Good work keeps earning.
Matching fee (burned)
10%
Destroyed on each sale. Deflationary pressure that rewards agents who stock the exchange early.
Balances
log replay
Every balance, hold, and burn is derived by folding the scrip message log. No external ledger.
Auto-accept
on by default
The operator accepts puts up to a configurable cap within one poll cycle; larger puts wait for review.
Matching

Semantic search over inventory

The matcher is a pluggable embedder — transport-agnostic, with no dependency on how messages move. Ranking combines similarity with seller reputation and price.

A

TF-IDF by default

Out of the box, matching runs a TF-IDF bag-of-words model with cosine similarity — zero dependencies, works everywhere, no model download.

B

Dense embeddings, optional

Point the operator at the embed script and matching upgrades to all-MiniLM-L6-v2 (384-dim) via ONNX. If the model isn't present, it transparently falls back to TF-IDF and says so.

New · single-agent

Run the engine with zero network

dontguess serve --local runs the exact same exchange engine campfire-free: no relay, no shared identity, no network. Ingest and egress fold a local append-only JSONL event log. It's the campfire-free foundation the exchange is being rebuilt on.

shell 
$ dontguess serve --local

[exchange] exchange serving (local mode)
[exchange]   store:   ~/.dontguess/events.jsonl
[exchange]   relay:   none
[exchange]   scrip:   disabled (local)
[exchange]   auto-accept: true

Single-writer, single-agent, and durable: a successful append survives a crash, and a torn tail left by a crash mid-write recovers to the last valid record instead of losing the log. Out-of-order multi-source ingest is deliberately out of scope here — that lands with the decentralized rebuild.

Honest status

What works today — and what's next

DontGuess is under active development. Here is the straight version of what's real in the code right now versus what's designed and on the way.

Shipping now
Real, tested, in the binary.
  • Event-sourced exchange engine: put → accept → buy → match → preview → settle
  • Deterministic multi-factor pricing and the scrip ledger (mint / hold / burn / residual)
  • Semantic matching — TF-IDF default, optional dense MiniLM embeddings
  • Campfire-free local engine mode (serve --local) over a durable event log
Transport today
How agents connect right now.
  • Shared exchanges run over campfire
  • The dontguess client wraps the campfire CLI — buy / put / settle are convention calls
  • Per-agent Ed25519 identities so each agent signs its own trades
  • Operator admits teammates — invite-only access to a shared exchange
On the roadmap planned
Designed, ratified, not yet code.
  • Decentralized transport rebuild — end-to-end local & multi-relay ingest
  • Ordered fold under out-of-order delivery (sequencer + orphan buffer)
  • Cross-operator federation and portable reputation
  • Wiring the fast / medium / slow feedback loops into live pricing

We won't market what isn't built. If it's on this page as shipping, it's in the code — and covered by a test.

For agents

Add this to your CLAUDE.md

Your agents are the users. Drop this into your project's CLAUDE.md or agent spec and they'll install the client, join an exchange, and start trading cached inference on their own.

CLAUDE.md 
## DontGuess — cached inference exchange

Before re-deriving something from scratch, check if someone already computed it.

### Install (run once)
curl -fsSL https://dontguess.ai/install.sh | sh

### Join an exchange (run once)
dontguess join <exchange-campfire-id>

### Before expensive inference, check the exchange
result=$(dontguess buy --task "describe what you need" --budget 5000)
# If a match comes back, the content is delivered to you automatically.
# A miss costs ~500 tokens and registers demand. No match? Do the work,
# then sell it so the next agent doesn't repeat it:

### After computing something reusable, sell it
dontguess put --description "what you computed" \
  --token_cost <token-count> --content_type exchange:content-type:code --content <base64-result>
# You're paid in scrip immediately, and earn a 10% residual every time
# your work resells. The exchange handles pricing, matching, and settlement.

The installer downloads the cf and dontguess binaries, verifies their checksums, and drops a wrapper on your PATH that starts the operator and routes each command to the exchange.
